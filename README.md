# 자리 배치 도구 (Seat Maker)

교사용 **자동 자리 배치** 웹 도구. 로그인 없이 두 화면(① 자리 설정 → ② 배치 결과)에서 끝나는 단일 페이지 앱입니다.
claude.ai/design 프로젝트 "교사용 자리 배치 도구"의 `자리배치도구.dc.html` 디자인을 실제 동작하는 React 앱으로 구현했습니다.

## 누구를 위한, 어떤 문제를 푸는 도구인가

**타겟 사용자**는 교사·강사입니다. 학기 초, 시험, 모둠활동 때마다 자리표를 새로 짜야 하는데, 엑셀이나 손으로 배치하면 세 가지가 늘 번거롭습니다.

- 사이가 나쁘거나 같이 두면 산만해지는 학생들을 **떨어뜨려 앉히기 어렵다**
- 조건이 바뀔 때마다 **매번 처음부터 다시 그려야 한다**
- 다 짜고 나면 **인쇄·배포용으로 포맷을 또 정리해야 한다**

Seat Maker는 이 세 가지를 한 번에 해결합니다. **로그인 없이 두 화면(① 자리 설정 → ② 배치 결과)** 만으로, *인접 금지* 같은 제약을 자동으로 만족하는 자리표를 즉시 만들고 A4로 인쇄합니다.

## 핵심 사용자 플로우

> **① 교실 설정** → 그리드·명단 입력, 기피 조합 지정 → **자리 생성** → **② 배치 결과** → 고정·드래그로 미세조정 → **PDF 인쇄**

설정 화면에서 우리 반 구조와 명단을 한 번 맞춰두면, "자리 생성하기" 한 번으로 제약을 만족하는 배치가 나옵니다. 결과가 마음에 안 들면 **고정·드래그·다시 배치**로 다듬고, 그대로 인쇄하면 끝입니다.

## 기능

### 교실을 그대로 옮기기 — *"실제 우리 반 구조를 화면에 재현"*

- **그리드 크기 조절**: 가로·세로 각각 2~10칸으로 우리 교실 좌석 배열에 맞춤
- **좌석 사용/미사용 토글**: 좌석 클릭으로 비는 자리(통로·결원)를 검은색으로 제외
- **짝꿍 모드**: 개별석 ↔ 2인 짝꿍석 배열 전환
- **교탁 표시**: 칠판·교탁 방향을 기준으로 자리표의 위/아래 방향을 직관적으로 인식

### 명단을 빠르게 채우기 — *"타이핑 없이 기존 명단 재활용"*

- **학생 추가/삭제·이름 편집**: 행 단위로 명단을 직접 구성
- **PDF 명단 자동 가져오기**: 기존 학생 명단 PDF를 올리면 번호·이름을 자동 추출합니다. 표형(엑셀 명단)과 자리표형(번호 위·이름 아래)을 모두 인식하고, **미리보기에서 확인·수정·삭제 후 적용**하므로 잘못 읽힌 항목도 바로 잡을 수 있습니다

### 제약을 만족하는 자동 배치 — *"산만함·갈등을 유발하는 조합을 자동으로 떨어뜨림"*

- **인접 금지(기피) 지정**: 각 학생에 떨어뜨릴 번호를 Enter로 태그 등록(최대 8명, **양방향 자동 동기화** — A가 B를 기피하면 B도 자동 기피)
- **8방향 인접 금지 자동 배치**: 상하좌우·대각선 어디로도 기피 조합이 붙지 않도록 자동 배치
- **솔직한 결과 안내**: 조건이 빡빡해 완벽 배치가 불가능하면 위반을 최소화한 배치를 내고, **어떤 쌍이 붙었는지**(예: 홍길동↔김영희)까지 알려줍니다

### 결과를 다듬고 배포하기 — *"자동 결과를 교사 의도대로 손보고 바로 출력"*

- **자리 고정**: 결과에서 자리 클릭 시 고정(노란 테두리), 다시 배치해도 위치 유지
- **드래그 수정**: 두 학생 자리를 드래그로 맞바꿈
- **다시 배치**: 고정석은 그대로 두고 나머지만 새로 배치
- **명렬표 토글 / 제목·하단 멘트 편집**: 자리표 옆 명단 표시 여부와 안내 문구를 상황에 맞게 조정
- **PDF 다운로드**: 편집한 제목·문구를 반영해 A4 가로로 인쇄·저장

## 신뢰성 약속

- **로그인·서버 없음**: 모든 작업(명단·그리드·배치·기피·제목/멘트)은 브라우저 `localStorage`에 자동 저장되어 새로고침해도 복원됩니다. 어떤 데이터도 서버로 전송되지 않습니다.
- **데이터로 깨지지 않음**: 손상되거나 버전이 다른 저장 데이터는 안전하게 무시하고 초기 상태로 시작합니다.

## 실행

```bash
npm install
npm run dev       # 개발 서버 (http://localhost:5173)
npm run build     # 타입체크 + 프로덕션 빌드
npm run preview   # 빌드 결과 미리보기
npm run test      # 단위 테스트 (Vitest)
npm run coverage  # 테스트 커버리지
```

## 구조

```
src/
├─ App.tsx                  앱 루트 (헤더 + 페이지 + 토스트)
├─ types.ts / constants.ts  도메인 타입 / 디자인 시스템 상수
├─ logic/                   순수 로직 (UI 비의존, 단위 테스트 대상)
│  ├─ seats.ts              좌석 키·8방향 인접 판정
│  ├─ avoid.ts              기피쌍·위반 검사
│  ├─ generate.ts           배치 휴리스틱
│  └─ layout.ts             그리드 스케일링 기하
├─ domain/                  도메인 규칙 (순수 함수)
│  ├─ student.ts            학생 생성 팩토리 (안정 id 부여)
│  └─ avoidRules.ts         기피 번호 검증·적용 규칙
├─ persistence/             localStorage 영속성 (zod 경계 검증)
│  ├─ schema.ts             직렬화 스키마 + 버전 관리
│  └─ storage.ts            저장/복원 (손상 시 안전 실패)
├─ hooks/                   useSeating(조립) + 도메인별 하위 훅
│  ├─ useGridConfig / useArrangement / useRoster
│  ├─ useEditableMeta / useToast / usePersistence
│  └─ useViewport
└─ components/              Header, SetupPage, ResultPage, SeatGrid,
                            StudentTable, RosterList, PrintArea,
                            TeacherDesk, ...
```

테스트는 `*.test.ts`로 소스 옆에 둡니다(logic·domain·persistence 계층 커버리지 80%+).
